# Plan 05-01: Canvas Visual Polish

**Phase:** 5 — Canvas Interaction Polish
**Requirements:** CANV-01, CANV-02, CANV-03
**Estimated Complexity:** Low
**Depends on:** Nothing (pure CSS + minor HTML changes)

---

## Goal
Polish three visual details on the canvas: simplify the workbench dialog background, increase grid density, and remove decorative corner pins from cards.

---

## Requirements Breakdown

| Req | Description | Current State | Target |
|-----|-------------|---------------|--------|
| CANV-01 | Workbench dialog background should tightly wrap the dialog, no extra blank frame | `.statusbar` spans full width with `background: var(--ps-blue)`; `.chatbar` is a 760px child centered inside | Move the blue background from `.statusbar` to `.chatbar`, so background only surrounds the dialog content. Remove `min-height` and full-width padding from `.statusbar`. |
| CANV-02 | Canvas grid density increase (~50%) | `.viewport` uses `background-size: 48px 48px` | Reduce `background-size` to `24px 24px` (exactly 50% denser). Verify visibility at default zoom 0.86. |
| CANV-03 | Remove decorative top-left corner dot from cards | Every `.node` has a `.pin` child (22px circle at `top: -12px; left: 24px`) | Remove `.pin` elements from DOM creation and from static HTML. Remove or deprecate `.pin`, `.pin-teal`, `.pin-gold` CSS rules. |

---

## Implementation Steps

### Step 1: Simplify workbench dialog background (CANV-01)

**Files:** `public/styles.css`, `public/index.html`

1. In `public/styles.css`:
   - `.statusbar`: remove `background: var(--ps-blue)`, `min-height: 112px`, and reduce padding to `0` (or minimal vertical padding for spacing). Keep `grid-area: main`, `align-self: end`, `display: flex`, `justify-content: center`, `z-index: 50`, `pointer-events: none`.
   - `.statusbar > *`: keep `pointer-events: auto`.
   - `.chatbar`: add `background: var(--ps-blue)`, `padding: 18px 30px 22px`, `border-radius: 24px 24px 0 0` (rounded top corners for dialog look), `box-shadow: 0 -12px 32px rgba(0, 0, 0, 0.1)` (move shadow here from `.statusbar`).

2. Verify the `.status-cluster` (status text + counts on the right) still renders correctly. It may need to be moved inside `.chatbar` or kept as-is if it was already a child. Check DOM structure in `public/index.html`.

3. Test: open the app, the blue background should only appear behind the chat input/dialog area, not stretch across the full viewport width.

### Step 2: Increase grid density (CANV-02)

**Files:** `public/styles.css`

1. In `.viewport` rule:
   - Change `background-size: 48px 48px;` to `background-size: 24px 24px;`

2. In `.placeholder-panel` rule (around line 940-947) there is another `background-size: 48px 48px;` for a placeholder background. Do NOT change this — it is unrelated to the canvas grid.

3. Test: open the app, zoom to default 0.86, grid lines should be clearly visible and roughly twice as dense as before.

### Step 3: Remove decorative corner pins (CANV-03)

**Files:** `public/styles.css`, `public/index.html`, `public/app.js`

1. In `public/index.html`:
   - Remove `<div class="pin pin-teal"></div>` from `#sourceNode`.
   - Remove `<div class="pin pin-gold"></div>` from `#analysisNode`.
   - In `<template id="optionTemplate">`, remove `<div class="pin"></div>`.

2. In `public/app.js`:
   - In `turnIntoGeneratedNode()`, remove the creation of `pin` element (lines 447-449).
   - Search for any other `.pin` element creation and remove.

3. In `public/styles.css`:
   - Remove `.pin` rule (lines 453-463).
   - Remove `.pin::after` rule (lines 466-472).
   - Remove `.pin-teal, .pin-gold` rule (lines 474-476).
   - Optionally keep the rules commented out for 1 commit in case of rollback, then fully remove in a follow-up.

4. Test: create or load a session, source node, analysis node, option nodes, and generated nodes should all have no top-left circular pin.
