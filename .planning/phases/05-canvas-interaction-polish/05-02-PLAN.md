# Plan 05-02: Node Collapse / Expand Interaction

**Phase:** 5 — Canvas Interaction Polish
**Requirements:** CANV-04
**Estimated Complexity:** Medium
**Depends on:** Nothing (builds on existing collapse system)

---

## Goal
Upgrade the node collapse/expand interaction so that:
1. After a user selects a direction and generates an image, unselected direction cards automatically collapse into their connection points.
2. Clicking a connection point toggles collapse/expand for the un-generated cards downstream of that node.
3. Double-clicking a connection point collapses all cards downstream of that node.
4. Triple-clicking a connection point expands all collapsed nodes in the entire tree.

---

## Current State Analysis

The app already has a collapse system in `public/app.js`:
- `state.collapsed: Set<string>` tracks collapsed node IDs.
- `ensureCollapseControl(id, element)` creates a `.collapse-dot` button on the left side of each node.
- `toggleCollapse(id)` adds/removes the node ID from `state.collapsed`.
- `applyCollapseState()` hides descendants of collapsed nodes with `collapsed-hidden` class.
- `updateCollapseControls()` refreshes button text/counts.
- `drawLinks()` skips links where either end is not visible.
- `anchor(node, side)` calculates connection coordinates; the right-side anchor (`side === "right"`) is at `node.x + width - 18`.

Current `.collapse-dot` behavior:
- Positioned at `left: -38px; top: 50%` (left edge of node).
- Single click toggles collapse.
- No double-click or triple-click handlers.
- No auto-collapse on generation.

Connection pins (the SVG circles drawn by `drawLinks()`):
- `pinA` at start of link (right side of from-node), `pinB` at end (left side of to-node).
- `r: 9`, class `link-pin`.
- Currently purely visual, no event listeners.

## Implementation Steps

### Step 1: Identify what to collapse after generation

When the user clicks "生成这张图" on an option node, the system calls `generateOption()` which eventually calls `turnIntoGeneratedNode()`. At that moment:
- The selected option node becomes a generated node.
- Its siblings (other option nodes sharing the same parent, i.e., the `analysis` node) should automatically collapse into their connection points on the parent.

Implementation:
1. In `generateOption()`, after `turnIntoGeneratedNode()` succeeds:
   - Find the parent of the selected option node by scanning `state.links` for `link.to === optionNodeId`.
   - Get all children of that parent via `getChildren(parentId)`.
   - Filter children to those that are NOT the selected node AND are NOT already generated (`!node.generated`).
   - For each such child, add its ID to a new auto-collapse tracking mechanism.

However, the requirement says "unselected direction cards automatically collapse **to the connection point**". This implies the parent node itself gets collapsed (hiding the unselected siblings), rather than collapsing each sibling individually. The visual effect is: the parent node's `.collapse-dot` shows a count of hidden siblings, and the unselected option nodes disappear.

Refined approach:
- After generation, add the **parent** node ID to `state.collapsed` if it has unselected, un-generated children.
- Then call `applyCollapseState()` and `updateCollapseControls()`.

This matches the existing collapse model: collapsing a node hides all its descendants.

### Step 2: Auto-collapse after generation

**File:** `public/app.js`

1. Locate `generateOption()` (search for `generateOption` or `turnIntoGeneratedNode`).
2. After the successful conversion to generated node:
   ```js
   const parentLink = state.links.find(l => l.to === optionNodeId);
   if (parentLink) {
     const siblings = getChildren(parentLink.from);
     const hasUnselected = siblings.some(sid => {
       const n = state.nodes.get(sid);
       return sid !== optionNodeId && n && !n.generated;
     });
     if (hasUnselected) {
       state.collapsed.add(parentLink.from);
       applyCollapseState();
       updateCollapseControls();
       autoSave();
     }
   }
   ```

### Step 3: Repurpose connection pins as interactive collapse toggles

The requirement says "单击连接点收纳该节点后未成图的卡片；双击收纳该节点后全部卡片；三击展开全部". The "connection point" is the visual dot where a link meets a node. We will attach event listeners to the SVG `circle` elements (`link-pin`) drawn by `drawLinks()`.

However, there is a subtlety: each link has two pins (`pinA` on the from-node, `pinB` on the to-node). The interaction should apply to the **to-node's** incoming connection pin (the left-side pin), because the user is collapsing/expanding the subtree **after** that node.

But wait — if a node is collapsed, its descendants are hidden, and links to hidden descendants are not drawn. So the pin would disappear. That's fine: when collapsed, the user sees the `.collapse-dot` on the node itself to expand.

Alternative interpretation: the user clicks the `.collapse-dot` on the node, not the link pin. The current `.collapse-dot` is already a button on the node. We can simply change its behavior:
- Single click: toggle collapse of un-generated descendants only.
- Double click: toggle collapse of ALL descendants.
- Triple click: expand all collapsed nodes globally.

This is simpler and more consistent, because `.collapse-dot` is already on every node, whereas link pins are transient SVG elements.

**Decision:** Enhance `.collapse-dot` click handlers rather than link pins, because:
1. `.collapse-dot` is a persistent DOM element on every node.
2. Link pins are recreated on every `drawLinks()` call, making event listener attachment fragile.
3. The requirement wording "连接点" could refer to the collapse dot (which is visually at the connection side of the node).

### Step 4: Implement click/double-click/triple-click on `.collapse-dot`

**File:** `public/app.js`

Replace the single `click` listener in `ensureCollapseControl()` with a multi-click detector.

1. Add a helper function:
   ```js
   function attachCollapseClickHandler(button, nodeId) {
     let clicks = 0;
     let timer = null;

     button.addEventListener("click", (event) => {
       event.stopPropagation();
       clicks++;
       if (clicks === 1) {
         timer = setTimeout(() => {
           // Single click: collapse/expand un-generated children only
           toggleCollapseUnGenerated(nodeId);
           clicks = 0;
         }, 280);
       } else if (clicks === 2) {
         clearTimeout(timer);
         timer = setTimeout(() => {
           // Double click: collapse/expand ALL children
           toggleCollapse(nodeId);
           clicks = 0;
         }, 280);
       } else if (clicks >= 3) {
         clearTimeout(timer);
         // Triple click: expand all collapsed nodes globally
         expandAllCollapsed();
         clicks = 0;
       }
     });
   }
   ```

2. Implement `toggleCollapseUnGenerated(nodeId)`:
   ```js
   function toggleCollapseUnGenerated(id) {
     const descendants = getDescendants(id);
     if (!descendants.size) return;

     // Check if there are any un-generated descendants currently visible
     const unGeneratedDescendants = [...descendants].filter(did => {
       const n = state.nodes.get(did);
       return n && !n.generated;
     });

     if (!unGeneratedDescendants.length) return;

     // Determine if we should collapse or expand
     // If ANY un-generated descendant is visible (not hidden by collapse), then collapse.
     const anyVisible = unGeneratedDescendants.some(did => {
       const n = state.nodes.get(did);
       return n && !n.element.classList.contains("collapsed-hidden");
     });

     if (anyVisible) {
       // We need to collapse the parent if not already collapsed
       if (!state.collapsed.has(id)) {
         state.collapsed.add(id);
       }
     } else {
       // All un-generated are already hidden; expand this parent
       state.collapsed.delete(id);
     }
     applyCollapseState();
     autoSave();
   }
   ```

   Wait — this logic is flawed because `applyCollapseState()` hides ALL descendants of a collapsed node, not just un-generated ones. The existing system does not support partial collapse (hiding only un-generated children while keeping generated ones visible).

   Options:
   A. Extend the collapse system to support "selective collapse" (hide only nodes matching a predicate).
   B. Change the interpretation: single-click on collapse dot toggles the existing full collapse, but only if there are un-generated children; if all children are generated, it does nothing.

   Re-reading the requirement: "单击连接点收纳该节点后未成图的卡片" — this literally means "clicking the connection point collapses the un-generated cards after this node". This implies selective collapse.

   To implement selective collapse cleanly, we should NOT use `state.collapsed` (which is binary per-node). Instead, introduce a new concept: `state.partialCollapse` — a Set of node IDs whose un-generated descendants should be hidden.

   But this adds complexity. A simpler approach:
   - Keep `state.collapsed` as-is for full collapse.
   - Add `state.hiddenNodes: Set<string>` for nodes individually hidden due to auto-collapse or single-click selective collapse.
   - `isNodeVisible()` checks both `collapsed-hidden` (from ancestor in `state.collapsed`) and a new `selective-hidden` class (from `state.hiddenNodes`).

   Actually, even simpler: when auto-collapsing after generation, we add the parent to `state.collapsed`. This hides ALL children (both generated and un-generated). But the generated child is the one the user just clicked, so it should remain visible. Therefore, auto-collapse must NOT collapse the parent if the generated child would also be hidden.

   Wait — in the current model, option nodes are children of the `analysis` node. When one option is generated, it becomes a `generated-node`. Its siblings are un-generated option nodes. If we collapse the `analysis` node, the generated node would also be hidden. That's bad.

   So auto-collapse should NOT collapse the parent. Instead, it should individually hide the un-selected sibling nodes.

   Let's re-think the model.

   **Revised model:**
   - `state.collapsed` remains for explicit user-driven full collapse (double-click).
   - `state.selectiveHidden: Set<string>` tracks nodes hidden by single-click or auto-collapse. These are individual node IDs, not ancestors.
   - `isNodeVisible(node)` returns false if the node is hidden by collapsed ancestor OR is in `selectiveHidden`.
   - `drawLinks()` skips links where either end is not visible.
   - `applyCollapseState()` applies both `collapsed-hidden` (from ancestors) and a new class `selective-hidden` (from `state.selectiveHidden`).

   **Auto-collapse after generation:**
   - Find siblings of the generated node.
   - Add each un-selected, un-generated sibling to `state.selectiveHidden`.
   - Call `applyCollapseState()`.

   **Single-click on collapse dot:**
   - If node has un-generated descendants that are visible: add them to `state.selectiveHidden`.
   - If all un-generated descendants are already hidden: remove them from `state.selectiveHidden` (expand).
   - Update the collapse dot appearance (show count of hidden un-generated descendants).

   **Double-click on collapse dot:**
   - Use existing `toggleCollapse()` (add/remove from `state.collapsed`). This hides ALL descendants.

   **Triple-click on collapse dot:**
   - Clear both `state.collapsed` and `state.selectiveHidden`.
   - Call `applyCollapseState()`.

   This model satisfies all requirements cleanly.

### Step 5: Implement the revised collapse model

**File:** `public/app.js`

1. Update state initialization:
   ```js
   const state = {
     // ... existing fields ...
     collapsed: new Set(),        // full collapse (double-click)
     selectiveHidden: new Set(),  // selective hide (single-click / auto-collapse)
     // ...
   };
   ```

2. Update `applyCollapseState()`:
   ```js
   function applyCollapseState() {
     for (const [id, node] of state.nodes.entries()) {
       const hiddenByAncestor = isHiddenByCollapsedAncestor(id);
       const selectivelyHidden = state.selectiveHidden.has(id);
       node.element.classList.toggle("collapsed-hidden", hiddenByAncestor);
       node.element.classList.toggle("selective-hidden", selectivelyHidden && !hiddenByAncestor);
     }
     updateCollapseControls();
     drawLinks();
   }
   ```
   Note: when a node is hidden by full collapse, it should not also have `selective-hidden` (redundant but harmless). The CSS should hide on either class:
   ```css
   .collapsed-hidden,
   .selective-hidden {
     display: none !important;
   }
   ```

3. Update `isNodeVisible()`:
   ```js
   function isNodeVisible(node) {
     return Boolean(node)
       && !node.element.classList.contains("hidden")
       && !node.element.classList.contains("collapsed-hidden")
       && !node.element.classList.contains("selective-hidden");
   }
   ```

4. Update `computeStateHash()` and `prepareStateForSave()` to include `selectiveHidden`.

5. Update load session logic to restore `selectiveHidden`.

6. Update `ensureCollapseControl()` click handler:
   ```js
   function ensureCollapseControl(id, element) {
     let button = Array.from(element.children).find((child) => child.classList?.contains("collapse-dot"));
     if (button) return button;

     button = document.createElement("button");
     button.type = "button";
     button.className = "collapse-dot";
     button.setAttribute("aria-label", "Collapse downstream nodes");
     attachMultiClickCollapseHandler(button, id);
     element.prepend(button);
     return button;
   }

   function attachMultiClickCollapseHandler(button, nodeId) {
     let clicks = 0;
     let timer = null;

     button.addEventListener("click", (event) => {
       event.stopPropagation();
       clicks++;
       if (clicks === 1) {
         timer = setTimeout(() => {
           toggleSelectiveCollapse(nodeId);
           clicks = 0;
         }, 280);
       } else if (clicks === 2) {
         clearTimeout(timer);
         timer = setTimeout(() => {
           toggleCollapse(nodeId);
           clicks = 0;
         }, 280);
       } else if (clicks >= 3) {
         clearTimeout(timer);
         expandAllCollapsed();
         clicks = 0;
       }
     });
   }
   ```

7. Implement `toggleSelectiveCollapse(id)`:
   ```js
   function toggleSelectiveCollapse(id) {
     const descendants = getDescendants(id);
     if (!descendants.size) return;

     const unGeneratedDescendants = [...descendants].filter(did => {
       const n = state.nodes.get(did);
       return n && !n.generated;
     });

     if (!unGeneratedDescendants.length) return;

     const anyVisible = unGeneratedDescendants.some(did => !state.selectiveHidden.has(did) && !isHiddenByCollapsedAncestor(did));

     if (anyVisible) {
       // Hide all un-generated descendants
       for (const did of unGeneratedDescendants) {
         state.selectiveHidden.add(did);
       }
     } else {
       // Show all un-generated descendants
       for (const did of unGeneratedDescendants) {
         state.selectiveHidden.delete(did);
       }
     }
     applyCollapseState();
     autoSave();
   }
   ```

8. Implement `expandAllCollapsed()`:
   ```js
   function expandAllCollapsed() {
     state.collapsed.clear();
     state.selectiveHidden.clear();
     applyCollapseState();
     autoSave();
   }
   ```

9. Update `updateCollapseControls()` to reflect both full and selective collapse counts:
   ```js
   function updateCollapseControls() {
     for (const [id, node] of state.nodes.entries()) {
       const button = Array.from(node.element.children).find((child) => child.classList?.contains("collapse-dot"));
       if (!button) continue;

       const descendants = getDescendants(id);
       const fullCollapsed = state.collapsed.has(id);

       // Count hidden nodes for badge display
       let hiddenCount = 0;
       for (const did of descendants) {
         if (state.collapsed.has(id) || state.selectiveHidden.has(did) || isHiddenByCollapsedAncestor(did)) {
           hiddenCount++;
         }
       }

       const hasChildren = descendants.size > 0;
       button.disabled = !hasChildren;
       button.textContent = (fullCollapsed || hiddenCount > 0) ? String(hiddenCount || descendants.size) : "";
       button.classList.toggle("is-collapsed", fullCollapsed || hiddenCount > 0);
       button.classList.toggle("has-children", hasChildren);
       button.title = hasChildren
         ? (fullCollapsed ? `展开 ${descendants.size} 个后续节点` : hiddenCount > 0 ? `展开 ${hiddenCount} 个后续节点` : `收起 ${descendants.size} 个后续节点`)
         : "没有后续节点";
       button.setAttribute("aria-label", button.title);
     }
   }
   ```

   Note: The count logic above may overcount if a descendant is hidden both by selective and by collapsed ancestor. A more precise count:
   ```js
   const hiddenCount = [...descendants].filter(did => {
     const n = state.nodes.get(did);
     return n && (n.element.classList.contains("collapsed-hidden") || n.element.classList.contains("selective-hidden"));
   }).length;
   ```

### Step 6: Auto-collapse siblings after generation

**File:** `public/app.js`

In `generateOption()`, after `turnIntoGeneratedNode()`:
```js
// Auto-collapse unselected siblings
const parentLink = state.links.find(l => l.to === optionNodeId);
if (parentLink) {
  const siblings = getChildren(parentLink.from);
  let hiddenAny = false;
  for (const sid of siblings) {
    if (sid === optionNodeId) continue;
    const n = state.nodes.get(sid);
    if (n && !n.generated) {
      state.selectiveHidden.add(sid);
      hiddenAny = true;
    }
  }
  if (hiddenAny) {
    applyCollapseState();
    autoSave();
  }
}
```

## Notes

- The 280ms click timeout is a common triple-click threshold. Adjust if it feels sluggish.
- `stopPropagation()` on the collapse dot prevents triggering node drag.
- Save/load must include `selectiveHidden` or the user will lose their collapsed state on refresh.
- The auto-collapse after generation only hides un-generated siblings. If the user later generates another sibling, that sibling becomes visible (because generated nodes are never in `selectiveHidden` by the logic above), and the remaining un-generated ones stay hidden.
