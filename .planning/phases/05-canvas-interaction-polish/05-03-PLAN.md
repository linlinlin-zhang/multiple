# Plan 05-03: Mouse Scroll Navigation

**Phase:** 5 — Canvas Interaction Polish
**Requirements:** CANV-05
**Estimated Complexity:** Low
**Depends on:** Nothing

---

## Goal
Change the canvas wheel behavior so that:
- Direct vertical scroll (no modifier key) pans the canvas up/down.
- Ctrl + vertical scroll zooms the canvas in/out.

---

## Current State Analysis

In `public/app.js` lines 156-160:
```js
viewport.addEventListener("wheel", (event) => {
  if (!event.ctrlKey && !event.metaKey) return;
  event.preventDefault();
  zoomBy(event.deltaY < 0 ? 0.06 : -0.06);
}, { passive: false });
```

Current behavior:
- Wheel without Ctrl/Meta: browser default (page scroll, but the viewport has `overflow: hidden`, so it does nothing).
- Wheel with Ctrl/Meta: zoom in/out by ±0.06.

Pan is currently only possible via mouse drag (pointerdown + pointermove on viewport).

Target behavior:
- Wheel without Ctrl/Meta: pan canvas vertically by `event.deltaY`. Optionally also pan horizontally by `event.deltaX` (for trackpad users).
- Wheel with Ctrl/Meta: zoom (same as current).

---

## Implementation Steps

### Step 1: Rewrite wheel event handler

**File:** `public/app.js`

Replace the wheel listener in `wireControls()` (around line 156):

```js
viewport.addEventListener("wheel", (event) => {
  event.preventDefault();
  if (event.ctrlKey || event.metaKey) {
    // Zoom
    zoomBy(event.deltaY < 0 ? 0.06 : -0.06);
  } else {
    // Pan
    state.view.x -= event.deltaX;
    state.view.y -= event.deltaY;
    updateBoardTransform();
  }
}, { passive: false });
```

Notes:
- `event.preventDefault()` is called unconditionally to stop browser default scrolling (especially important for trackpad gestures).
- Pan deltas are subtracted because scrolling "down" (positive deltaY) should move the canvas content down, which means the view origin moves up — i.e., `view.y` decreases. Wait, let's verify:
  - Current drag logic: `state.view.y = panStart.viewY + event.clientY - panStart.y`. If I drag the mouse down (increase Y), `view.y` increases. This means the board moves down relative to the viewport, which feels like the content follows the cursor. That's standard "grab and pan".
  - For wheel: if I scroll down (positive deltaY), the standard expectation is that the canvas content moves up (like scrolling a page). So `view.y` should decrease. Therefore `state.view.y -= event.deltaY` is correct.
  - Similarly, scroll right (positive deltaX) should move content left: `state.view.x -= event.deltaX`.

- Trackpad users often generate both `deltaX` and `deltaY`. Handling both provides natural 2D panning.

### Step 2: Ensure no conflicts with other wheel listeners

Search the entire `public/app.js` for other `wheel` listeners. There should be only the one in `wireControls()`.

If any node inside the viewport (e.g., a scrollable textarea or dropdown) needs wheel events, `event.stopPropagation()` on those inner elements would be needed, but currently there are no such elements inside nodes.

### Step 3: Optional — Add shift-scroll as horizontal pan

Some users expect Shift+scroll to pan horizontally. The current implementation already handles `deltaX` natively (trackpads). For wheel mice that only emit `deltaY`, we could add:

```js
let dx = event.deltaX;
let dy = event.deltaY;
if (event.shiftKey && Math.abs(dy) > 0 && dx === 0) {
  dx = dy;
  dy = 0;
}
state.view.x -= dx;
state.view.y -= dy;
```

This is optional but improves accessibility for mouse-wheel-only users. Include it in the plan as a nice-to-have.

### Step 4: Test zoom range

The existing `zoomBy()` clamps scale to `[0.45, 1.35]`. No change needed.

## Notes

- `{ passive: false }` is required because we call `event.preventDefault()`.
- The delta values from `wheel` events can vary wildly between browsers and input devices. The raw subtraction should feel reasonable because the transform is applied in screen pixels; no scaling by `state.view.scale` is needed for pan.
- If panning feels too fast or too slow with high-resolution mouse wheels, consider a damping factor (e.g., `state.view.y -= event.deltaY * 0.5`). Start with 1.0 and adjust based on feel.
