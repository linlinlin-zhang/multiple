  <action>
Add a new endpoint `POST /api/analyze-url` in server.js, placed after `/api/analyze-text`.

Implementation:
1. **Route handler**:
   ```javascript
   if (req.method === "POST" && url.pathname === "/api/analyze-url") {
     const body = await readJson(req);
     return await handleAnalyzeUrl(body, res);
   }
   ```

2. **Implement `handleAnalyzeUrl(body, res)`**:
   - Accept `body.url` (string). Validate it is a valid HTTP/HTTPS URL using `new URL()`.
   - If invalid or not http/https, return 400 `{ error: "Invalid URL. Only http:// and https:// links are supported." }`.
   - Extract domain for display: `new URL(body.url).hostname`.
   - If `isDemoRole(ANALYSIS_CONFIG)`, return `buildDemoAnalysis(domain)`.
   - Build prompt instructing the model to search/summarize the URL and generate creative directions:
     ```
     你是一个网络内容创意导演，正在为一个画布式图片生成应用分析用户提供的网页链接。
     请基于你对该网页内容的理解和搜索能力，总结其核心主题、视觉氛围、可延展的叙事方向，并给出 5 个不同的成图方向。
     这些方向会作为画布上的分支节点展示，用户点击后会调用成图模型。
     请只返回严格 JSON，不要 Markdown，不要代码块。

     JSON 结构：{ title, summary, detectedSubjects, moodKeywords, options[...] }
     （与图像分析完全一致的结构）

     要求：方向之间要明显不同；不要生成暴力、色情、仇恨或侵犯隐私的内容。

     网页链接：${body.url}
     网页域名：${domain}

     请尽可能基于该网页的内容主题生成视觉方向。如果无法访问该链接，请基于域名和常见内容类型给出合理的创意推测。
     ```
   - Call `chatCompletions(ANALYSIS_CONFIG, { messages: [{ role: "user", content: prompt }] })`.
   - Parse with `parseJsonFromText`, normalize with `normalizeAnalysis(parsed, domain)`.
   - Return JSON with `provider`, `model`, `title`, `summary`, `detectedSubjects`, `moodKeywords`, `options`, and `domain`.

3. **No server-side HTTP fetch to the URL** — per D-10, rely on the AI model's search/summary capability rather than scraping. This avoids:
   - Handling JavaScript-rendered pages
   - Bot protection / CAPTCHAs
   - Large HTML parsing
   - Security risks from fetching arbitrary user URLs

4. **URL validation security**:
   - Reject `file://`, `ftp://`, `javascript:`, `data:` protocols.
   - Reject private IP ranges (10.x, 192.168.x, 127.x, 172.16-31.x) to prevent SSRF.
   - Reject localhost and loopback.
   - Add a helper `isValidPublicUrl(urlString)` that performs these checks.

3. Implement `analyzeUrl()`:
   ```javascript
   async function analyzeUrl() {
     const url = urlInput.value.trim();
     if (!url) return;
     setStatus("Analyzing URL", "busy");
     urlAnalyzeButton.disabled = true;

     try {
       const data = await postJson("/api/analyze-url", { url });
       state.sourceType = "url";
       state.sourceUrl = url;
       state.fileName = new URL(url).hostname;
       state.latestAnalysis = data;

       // Render source preview as a link card
       renderUrlSource(url, data.title);
       renderAnalysis(data);
       renderOptions(data.options || []);
       setStatus("Branches ready", "ready");
       autoSave();
     } catch (error) {
       setStatus(error.message || "URL analysis failed", "error");
     } finally {
       urlAnalyzeButton.disabled = false;
     }
   }
   ```

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-06-06 | Spoofing | Malicious URL input | mitigate | `isValidPublicUrl()` rejects non-http(s), private IPs, localhost, loopback |
| T-06-07 | Denial of Service | Very long URL | mitigate | Validate URL length <= 2048 chars; MAX_BODY_BYTES limits request size |
| T-06-08 | Information Disclosure | URL sent to AI API | accept | URL is user-provided content for analysis; no PII expected |
| T-06-09 | Elevation of Privilege | SSRF via URL | mitigate | No server-side fetch to the URL; only the AI model sees it. Private IP / localhost filters prevent accidental exposure if fetch is added later. |
</threat_model>
