## Existing node structure (from public/app.js)
Nodes are DOM elements with class `.node`. Source node has `.source-node`, analysis `.analysis-node`, options `.option-node`, generated `.generated-node`.

Existing collapse/expand uses multi-click on `.collapse-dot` buttons (single-click selective collapse, double-click full collapse, triple-click expand all). The `.collapse-dot` is prepended inside each node element.

`registerNode(id, element, data)` stores node in `state.nodes` Map and calls `ensureCollapseControl(id, element)` which prepends the collapse dot.
`makeDraggable(element, id)` attaches `pointerdown`/`pointermove`/`pointerup` for dragging.

## Key CSS variables available
- `--ps-blue: #0070cc` (light) / `#4aa8ff` (dark) — theme color for selection outline
- `--ring: 0 0 0 2px #ffffff, 0 0 0 4px var(--ps-blue)` — existing focus ring pattern

    1. Add `.node.is-selected` rule:
       ```css
       .node.is-selected {
         outline: 2px solid var(--ps-blue);
         outline-offset: 4px;
         border-radius: 24px; /* match node border-radius */
         z-index: 9;
       }
       ```
       Note: `outline` is used instead of `border` to avoid affecting layout/size. `outline-offset: 4px` creates the "surrounding" effect. `border-radius` ensures the outline follows the card's rounded corners.
    2. Ensure `.node.dragging` has `z-index: 8` (already exists). Selected (`z-index: 9`) should sit above unselected but below dragging during active drag. This is acceptable because drag adds `.dragging` temporarily; after `pointerup`, `.dragging` is removed and the selected outline remains visible at z-index 9.
    3. Dark mode compatibility: `var(--ps-blue)` already switches to `#4aa8ff` in dark mode via `html[data-theme="dark"]` root variables, so no extra dark rule needed.
    4. Add transition for smooth outline appearance:
