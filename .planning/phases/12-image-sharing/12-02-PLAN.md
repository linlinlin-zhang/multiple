<task type="auto">

  <files>server.js, src/api/sessions.js</files>
  <read_first>
    <file>server.js</file>
    <file>src/api/sessions.js</file>
  </read_first>
  <action>
1. In `server.js`, add `POST /api/share-image` endpoint:
   - Accepts: `{ nodeId, sessionId }`
   - Looks up the generated node in the session
   - Creates a `ShareToken` record (or reuse existing ShareToken model) with:
     - token: random UUID
     - type: "image"
     - nodeData: serialized node data (imageHash, explanation, option data, source summary)
     - expiresAt: optional (default 30 days)
   - Returns: `{ shareUrl: "/share-image/{token}" }`
2. Add `GET /share-image/:token` endpoint:
   - Looks up token
   - Returns HTML page (or JSON for API separation)
   - If token not found, return 404
3. Reuse existing `ShareToken` Prisma model if it exists, or create minimal in-memory storage if DB unavailable
  </action>
  <verify>
    <automated>grep -n "/api/share-image\|/share-image/" server.js</automated>
  </verify>
  <acceptance_criteria>
    - POST /api/share-image creates share token
    - GET /share-image/:token returns share data
    - 404 for invalid tokens
  </acceptance_criteria>
  <done>Share-image API endpoints implemented.</done>
</task>

  <action>
1. Implement `handleShareImage(nodeId)`:

   - Calls `POST /api/share-image` with `{ nodeId, sessionId }`
   - Receives `{ shareUrl }`
   - Copies URL to clipboard: `navigator.clipboard.writeText(window.location.origin + shareUrl)`
   - Shows toast: "分享链接已复制到剪贴板" / "Share link copied to clipboard"
