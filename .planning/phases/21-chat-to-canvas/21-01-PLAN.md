# Plan 01: Whitelist Reconciliation

## Objective
Eliminate the whitelist mismatch between `server.js` and `src/prompts/shared.js` by replacing the hardcoded `CANVAS_TOOL_TYPES` array in server.js with an import from the prompt module's `CANVAS_ACTION_TYPES`. This unblocks all 7 rich node types (`create_note`, `create_plan`, `create_todo`, `create_weather`, `create_map`, `create_link`, `create_code`) and 4 zoom actions (`zoom_in`, `zoom_out`, `set_zoom`, `reset_view`) that the LLM is already instructed to emit but which are currently silently filtered out by `normalizeVoiceActions`.

- `CANVAS_TOOL_TYPES` hardcoded array is removed from `server.js`
- `VOICE_ACTION_TYPES` Set is built from the imported array
- All existing action types continue to work (zero regression)
- Frontend `executeCanvasAction` already supports the newly-whitelisted types â€” no frontend changes needed
