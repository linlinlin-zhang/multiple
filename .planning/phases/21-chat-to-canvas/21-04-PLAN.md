# Plan 04: Markdown Rendering

## Objective
Render assistant chat messages as sanitized HTML using micromark + DOMPurify. Replace the current plain-text `textContent` rendering with full markdown support: headings, lists, code blocks (with language fences), links, tables, bold/italic. Add a copy button to each code block. The app is vanilla JS with no frontend build step, so dependencies load via CDN.

## must_haves
- Assistant messages render as sanitized HTML from markdown
- Code blocks have a "Copy" button
- XSS attempts are neutralized by DOMPurify
- Streaming markdown progressively renders without full re-parses causing flicker
- Chinese and English content both render correctly
- Existing plain-text user messages remain unaffected

### Task 2: Create markdown render helper

<read_first>
- public/app.js (lines 5715-5740)
</read_first>

<action>
In `public/app.js`, add a helper function after the existing utility functions (or near `renderChatMessage`):

```js
function renderMarkdownToHtml(markdown) {
  if (!markdown) return "";
  const rawHtml = micromark(markdown);
  return DOMPurify.sanitize(rawHtml, {
    ALLOWED_TAGS: [
      "p", "h1", "h2", "h3", "h4", "h5", "h6",
      "ul", "ol", "li", "strong", "em", "code", "pre",
      "blockquote", "table", "thead", "tbody", "tr", "th", "td",
      "a", "br", "hr", "div", "span"
    ],
    ALLOWED_ATTR: ["href", "target", "rel", "class"],
    ALLOW_DATA_ATTR: false
  });
}

function addCopyButtons(container) {
  container.querySelectorAll("pre code").forEach((codeBlock) => {
    const pre = codeBlock.parentElement;
    if (pre.querySelector(".chat-code-copy")) return; // already has button
    const button = document.createElement("button");
    button.className = "chat-code-copy";
    button.textContent = t("chat.copyCode") || "Copy";
    button.setAttribute("aria-label", t("chat.copyCode") || "Copy code");
    button.onclick = () => {
      navigator.clipboard.writeText(codeBlock.textContent).then(() => {
        button.textContent = t("chat.copied") || "Copied!";
        setTimeout(() => {
          button.textContent = t("chat.copyCode") || "Copy";
        }, 1500);
      }).catch(() => {});
    };
    pre.appendChild(button);
  });
}
```

Add translation keys to the i18n object:
```js
"chat.copyCode": "复制",
"chat.copied": "已复制"
// English versions in en block:
"chat.copyCode": "Copy",
"chat.copied": "Copied!"
```
</action>

<acceptance_criteria>

- `grep -n 'chat.copyCode' public/app.js` returns translation key definitions in both zh and en blocks
- `grep -n 'DOMPurify.sanitize' public/app.js` returns the sanitize call

  const text = document.createElement("span");

// AFTER
if (message.content) {
  const text = document.createElement("span");
  text.className = "chat-text markdown-body";
  if (message.role === "assistant") {
    text.innerHTML = renderMarkdownToHtml(message.content);

    text.textContent = message.content;
  }
  line.appendChild(text);
}
```

  content: replyBuffer,

  textEl.innerHTML = renderMarkdownToHtml(message.content);

```

Caution: Re-parsing full markdown on every delta could be slow for long replies. For Phase 21, this is acceptable — micromark is fast. If performance becomes an issue, optimize in a future phase by only re-parsing changed suffix.

Ensure the streaming cursor (from Plan 03) is appended AFTER the rendered HTML, not inside it.
</action>
