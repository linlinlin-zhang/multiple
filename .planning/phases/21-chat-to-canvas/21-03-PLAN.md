# Plan 03: Streaming Default

## Objective
Make SSE streaming the default for all chat modes, not just thinking mode. Currently `public/app.js` only calls `postStreamingChat` when `effectiveThinkingMode === "thinking"`; no-thinking mode uses blocking `postJson`. The backend already supports streaming (`server.js:1058` checks `body?.stream === true`). This plan enables progressive character rendering for the default chat experience.

## must_haves
- All chat requests include `stream: true` in payload
- No-thinking mode uses `postStreamingChat` instead of `postJson`
- Frontend handles `reply` SSE events (not just `thinking` events)
- Streaming cursor is visible during reply generation
- Final event contains complete reply + actions
- Voice/realtime chat also streams

```js
// Inside postStreamingChat SSE parser

event: thinking → accumulate reasoning delta (existing behavior)
```

1. Append each delta to a running buffer
2. Update the pending assistant message's `content` with the buffer
3. Re-render the message DOM (or update text node directly for performance)

```js

  updateChatMessage(pendingAssistant, { content: replyBuffer });
}
```

</action>
