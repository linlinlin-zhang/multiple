# Plan 02: Tool Calling Refactor

## Objective
Replace the dual-prompt JSON orchestrator architecture with OpenAI-style tool calling. The chat endpoint will send a conversational system prompt plus a `tools` array describing canvas actions. The LLM returns free-form markdown in `message.content` and structured canvas actions in `message.tool_calls`. This removes the hard "1-3 sentences" length constraint, eliminates forced JSON object mode, and allows the model to adapt reply length to user intent.

## must_haves
- Chat endpoint uses OpenAI `tools` + `tool_choice: "auto"` instead of forced JSON schema
- `message.content` is treated as free-form markdown reply (no length cap)
- `message.tool_calls` is parsed into canvas action objects
- Fallback inference covers all 7 rich node types plus zoom when `tool_calls` is empty/malformed
- Default `max_tokens` for chat role is â‰¥ 4096
- All hard length constraints removed from chat prompts
- Server starts and chat endpoint responds correctly

  return toolCalls

function ensureChatFallbackActionsClean(message, actions, reply = "") {
  const normalized = Array.isArray(actions) ? [...actions] : [];
  if (normalized.length > 0) return normalized;

  const requestText = String(message || "").normalize("NFKC");
  const combinedText = requestText + " " + String(reply || "").normalize("NFKC");

  for (const [actionType, regex] of Object.entries(FALLBACK_KEYWORDS)) {
    if (regex.test(combinedText)) {

      normalized.push({ type: actionType, title, description: reply || title });
      break; // only one fallback action per message
    }
  }
  return normalized;
