Create `src/api/import.js` with `handleImportSession(body, res)`.

Implementation steps:
1. Validate `body` is an object. If not, return 400 `{ error: "Invalid request body" }`.
2. Validate `body.version` is `1`. If missing or not `1`, return 400 `{ error: "Unsupported export version" }`.
3. Extract `session = body.session`. Validate it has `nodes` (array), `links` (array), and `chatMessages` (array). If not, return 400 `{ error: "Invalid session data" }`.
4. Extract `assets = body.assets || []`.
5. For each asset with `dataUrl` present (non-null string starting with `data:`), store it via `storeDataUrl(asset.dataUrl, { kind: asset.kind })`. Capture the returned `hash`. If the stored hash differs from the asset's original hash (possible if file was re-encoded), use the NEW hash for the database record. If `storeDataUrl` throws, log the error and skip that asset (it will not be in the restored session).
   ```javascript
   const storedAssets = [];
   for (const asset of assets) {
     if (typeof asset.dataUrl === "string" && asset.dataUrl.startsWith("data:")) {
       try {
         const stored = await storeDataUrl(asset.dataUrl, { kind: asset.kind });
         storedAssets.push({
           hash: stored.hash,
           kind: asset.kind,
           mimeType: asset.mimeType || stored.mimeType,
           fileSize: stored.size,
           fileName: asset.fileName || null
         });
       } catch (err) {
         console.error("[import] failed to store asset:", err.message);
       }
     } else if (asset.hash) {
       // Asset metadata without dataUrl — keep reference but file may be missing
       storedAssets.push({
         hash: asset.hash,
         kind: asset.kind,
         mimeType: asset.mimeType || "image/jpeg",
         fileSize: asset.fileSize || 0,
         fileName: asset.fileName || null
       });
     }
   }
   ```
6. Create the new session and all related rows inside a Prisma transaction:
   ```javascript
   const newSession = await prisma.$transaction(async (tx) => {
     const sessionRecord = await tx.session.create({
       data: {
         title: session.title || "导入的会话",
         isDemo: session.isDemo || false,
         viewState: session.viewState || { x: 0, y: 0, scale: 0.86 }
       }
     });

     if (storedAssets.length) {
       await tx.asset.createMany({
         data: storedAssets.map(a => ({ ...a, sessionId: sessionRecord.id }))
       });
     }

     if (session.nodes?.length) {
       await tx.node.createMany({
         data: session.nodes.map(n => ({
           sessionId: sessionRecord.id,
           nodeId: n.nodeId,
           type: n.type,
           x: n.x,
           y: n.y,
           width: n.width || 318,
           height: n.height || 220,
           data: n.data || {},
           collapsed: n.collapsed || false
         }))
       });
     }

     if (session.links?.length) {
       await tx.link.createMany({
         data: session.links.map(l => ({
           sessionId: sessionRecord.id,
           fromNodeId: l.fromNodeId,
           toNodeId: l.toNodeId,
           kind: l.kind || "option"
         }))
       });
     }

     if (session.chatMessages?.length) {
       await tx.chatMessage.createMany({
         data: session.chatMessages.map(m => ({
           sessionId: sessionRecord.id,
           role: m.role,
           content: m.content
         }))
       });
     }

     return sessionRecord;
   });
   ```
7. Return 200 JSON:
   ```javascript
   {
     ok: true,
     sessionId: newSession.id,
     title: newSession.title,
     createdAt: newSession.createdAt
   }
   ```

Use the same `sendJson` helper as other API files. Import `prisma` from `../lib/prisma.js` and `storeDataUrl` from `../lib/storage.js`.

   document.getElementById("exportBtn")?.addEventListener("click", async () => {
     if (!currentSessionId) {
       alert("请先保存会话后再导出。");
       return;
     }
     try {
       const res = await fetch(`/api/sessions/${currentSessionId}/export`);
       if (!res.ok) throw new Error(await res.text());
       const blob = await res.blob();
       const url = URL.createObjectURL(blob);
       const a = document.createElement("a");
       a.href = url;
       // Extract filename from Content-Disposition if present

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| client -> import API | Untrusted client uploads arbitrary JSON; must validate structure before DB write |
| import API -> storage | Base64 images decoded and written to disk; must validate format and size |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-07 | Tampering | handleImportSession payload | mitigate | Validate `version === 1`, require `session.nodes`, `session.links`, `session.chatMessages` to be arrays. Reject unexpected top-level keys. Prisma createMany uses parameterized queries. |
| T-03-08 | Denial of Service | handleImportSession large payload | mitigate | `readJson` in server.js already enforces `MAX_BODY_BYTES = 22 * 1024 * 1024` (22 MB). Base64 images are bounded by this limit. Accept because single-user local use makes large-scale abuse unlikely. |
| T-03-09 | Information Disclosure | handleExportSession | accept | Only exports sessions by ID. No auth in v1, but session IDs are UUIDs (unguessable). Accept because this matches existing session access model. |
| T-03-10 | Elevation of Privilege | handleImportSession creates session | accept | Any client can create a session via import. Accept because there is no user auth or privilege model in v1; all sessions are equally accessible. |
</threat_model>
