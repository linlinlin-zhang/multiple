<task type="auto">
  <name>Task 2: Fix edge cases and ensure transaction safety</name>
  <files>src/api/sessions.js, public/app.js</files>
  <action>
Review and fix the following edge cases:

1. **Transaction boundary audit in `src/api/sessions.js`**:
   - Ensure `prisma.$transaction` wraps ALL database writes in both create and update
   - File writes (images) MUST happen BEFORE the transaction starts, not inside it
   - If a file write succeeds but the transaction fails, the file is orphaned. For Phase 1, accept this risk (documented in PITFALLS.md as acceptable for v1). Add a TODO comment: `// TODO: orphaned file cleanup job`

2. **Empty state handling**:
   - `handleCreateSession` must accept `state.nodes` as either a Map (from frontend) or a plain object
   - `handleGetSession` must return nodes in an order that reconstructs the graph correctly

3. **Asset deduplication verification**:
   - Upload the same image twice via `/api/assets` -> verify the same hash is returned and only one file exists on disk

4. **Demo mode flag propagation**:
   - When the server is in demo mode (`modeBadge === "demo"`), `isDemo` must be `true` on the Session
   - Verify `GET /api/history?includeDemo=false` excludes demo sessions
   - Verify `GET /api/history?includeDemo=true` includes them

5. **Auto-save debounce correctness**:
   - Rapidly drag a node 10 times -> only one save should fire after 2 seconds of inactivity
   - Verify by checking server logs or saveStatus text

6. **Session restoration fidelity**:
   - After loading a session, `state.nodes` must have the same `x, y, width, height` values
   - Generated nodes must display the correct image (via `/api/assets/:hash?kind=generated`)
   - Chat messages must be in the correct order
   - Collapsed state must be preserved

7. **URL parameter handling**:
   - If `?session=xxx` is present but the session doesn't exist, show an error and clear the param
   - Add in `init()`:
```javascript
const urlParams = new URLSearchParams(window.location.search);
const resumeSessionId = urlParams.get("session");
if (resumeSessionId) {
  loadSession(resumeSessionId).catch(() => {
    const url = new URL(window.location.href);
    url.searchParams.delete("session");
    window.history.replaceState({}, "", url);
    setStatus("SESSION NOT FOUND", "error");
  });
}
```
