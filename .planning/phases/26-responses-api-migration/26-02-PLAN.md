# Phase 26 Plan 02 — Real API Smoke Tests & Compatibility Fixes

## Goal

Validate the Phase 26 Responses API migration against the real DashScope API and fix compatibility gaps discovered by live requests.

## Scope

1. Start the server with the real `.env` configuration on an isolated test port.
2. Run a minimal non-search chat request to verify `/responses` payload shape.
3. Run a canvas-action request to verify custom `canvas_action` tool calling.
4. Run a multi-turn request to verify `previous_response_id` is returned and accepted.
5. Run a web-search request to verify builtin `web_search` and reference extraction.
6. Add lightweight smoke-test tooling if manual commands are too brittle.
7. Record verified behavior and unresolved API quirks in this plan.

## Verification Matrix

| Check | Expected |
|-------|----------|
| Health | `/api/health` returns `mode: api` for chat |
| Minimal chat | `/api/chat` returns assistant reply without schema error |
| Canvas tool | Trip/planning prompt returns executable `actions` |
| Multi-turn | First response returns `responseId`; second request with `previousResponseId` succeeds |
| Web search | Search prompt returns `references` and/or reference artifacts |
| Static checks | `node --check server.js`, `node --check public/app.js`, `git diff --check` pass |

## Risk Controls

- Use short prompts and low request count to control API cost.
- Use an isolated port so existing dev servers are not disturbed.
- Never print API keys or `.env` values.
- Stop test server after smoke tests.

## Results

- Health check passed on isolated port `3027` with `chat: api`.
- Minimal streaming chat passed and returned `responseId`.
- Multi-turn chat passed: a second request with `previousResponseId` reused prior context.
- Canvas tool passed: a Japan 7-day trip prompt returned one executable `create_plan` action after duplicate tool-call de-duping.
- Web search passed: a Qwen3 official-information prompt returned structured `references` and reference artifacts.
- Citation normalization added: `[1]`-style model citations are normalized to `[ref_1]` for frontend rendering.
- Code interpreter passed for a minimal Python calculation after enabling built-in-tool thinking and suppressing `canvas_action` for explicit result-only code requests.
- Static checks passed: `node --check server.js`, `node --check public/app.js`, and `git diff --check`.

## Compatibility Fixes Applied

- Web search is now opt-in by explicit search/URL/current-info intent instead of defaulting on for normal chat.
- Removed fallback fake Google `web_search` actions when no real references are returned.
- Streaming reasoning deltas are no longer emitted as assistant reply deltas.
- Built-in tools force `enable_thinking: true` to match DashScope Responses examples.
- Duplicate custom tool calls from streamed and completed Responses payloads are de-duplicated by function name and arguments.
- Empty code-interpreter artifacts without output text or URL are filtered out.
