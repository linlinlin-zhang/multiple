# Phase 26 Plan 03 — Frontend UI Smoke Test

## Goal

Verify the Responses API migration through the real browser UI, not only direct API calls.

## Scope

1. Start the app with the real `.env` configuration on a dev port.
2. Open the browser preview and verify the chat UI loads.
3. Send a minimal chat prompt and confirm the assistant message appears.
4. Send a follow-up prompt in the same thread and confirm multi-turn continuity.
5. Send a web-search prompt and confirm `[ref_n]` citations render as clickable links with reference/artifact cards.
6. Send a canvas-action prompt and confirm a canvas node is created and the chat action result is shown.
7. Send a result-only Python calculation prompt and confirm no unwanted canvas action is created.
8. Record UI issues and fixes.

## Verification Matrix

| Check | Expected |
|-------|----------|
| App load | Canvas and chat sidebar load without console errors |
| Minimal chat | Assistant reply streams and finalizes |
| Thread continuity | Follow-up uses the same `previousResponseId` chain |
| Citations | `[ref_n]` markers become links when references exist |
| Artifacts | Reference/material cards render without empty code artifacts |
| Canvas action | Trip/planning prompt creates one structured plan node |
| Code-only prompt | Result-only Python prompt returns text only, no canvas action |
