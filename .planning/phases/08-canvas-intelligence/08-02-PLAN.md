## Key interfaces from codebase

Generated node structure (from `turnIntoGeneratedNode()` in `public/app.js`):
```typescript
function turnIntoGeneratedNode(element, option, imageDataUrl) {
  element.className = "node option-node generated-node";
  element.innerHTML = "";
  ensureCollapseControl(element.dataset.nodeId, element);

  const img = document.createElement("img");
  img.className = "generated-image";
  img.src = imageDataUrl;
  img.alt = option.title || "生成图";
  element.appendChild(img);
  // ... eyebrow, title, description, actions
}
```

Node data available in `state.nodes.get(id)`:
```typescript
interface Node {
  id: string;
  element: HTMLElement;
  x: number; y: number; width: number; height: number;
  generated?: boolean;
  option?: object;
  imageHash?: string;
  explanation?: string;
}
```

Images are served at `/api/assets/{hash}?kind=generated`.

Existing modal pattern: `.settings-panel` and `.session-panel` use fixed-position slide-out panels with `transform: translateX(100%)` for hiding. The image viewer should use a centered overlay modal pattern instead (different UX).

2. Implement `openImageViewer(nodeId)`:
   ```js
   function openImageViewer(nodeId) {
     const node = state.nodes.get(nodeId);
     if (!node || !node.generated) return;

     const img = node.element.querySelector(".generated-image");
     if (!img) return;

     viewerImage.src = img.src;
     viewerImage.alt = node.option?.title || "生成图";
     viewerTitle.textContent = node.option?.title || "";
     viewerExplanation.textContent = node.explanation || "";

     imageViewerModal.classList.remove("hidden");
     document.body.style.overflow = "hidden";

     // Focus trap: focus the close button
     const closeBtn = imageViewerModal.querySelector("#closeImageViewer");
     if (closeBtn) closeBtn.focus();
   }
   ```

3. Implement `closeImageViewer()`:
   ```js
   function closeImageViewer() {
     imageViewerModal.classList.add("hidden");
     viewerImage.src = "";
     document.body.style.overflow = "";
   }
   ```

4. Wire event listeners in `wireControls()` (or in a dedicated `wireImageViewer()` called from `init()`):
   ```js
   // Close on backdrop click
   imageViewerModal?.querySelector("[data-close-modal]")?.addEventListener("click", closeImageViewer);
   // Close on X button
   document.querySelector("#closeImageViewer")?.addEventListener("click", closeImageViewer);
   // Close on Escape key
   document.addEventListener("keydown", (event) => {
     if (event.key === "Escape" && !imageViewerModal?.classList.contains("hidden")) {
       closeImageViewer();
     }
   });
   ```

5. Wire generated node images to open the modal:
   - In `turnIntoGeneratedNode()`, after creating `img`, add:

However, `loadSession` also manually rebuilds nodes from persisted state. Ensure the click listener is attached in all code paths that create generated node DOM.
   - Verify: `turnIntoGeneratedNode` is the single place that builds the generated node DOM. Both `generateOption()` and `loadSession()` call it, so adding the listener there covers both paths.
