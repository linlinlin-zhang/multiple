## Key interfaces from codebase

Node shape (from `state.nodes` Map values):
```typescript
interface Node {
  id: string;
  element: HTMLElement;
  x: number;
  y: number;
  width: number;
  height: number;
  generated?: boolean;
  option?: object;
  imageHash?: string;
  explanation?: string;
}
```

Link shape (from `state.links`):
```typescript
interface Link {
  from: string;
  to: string;
  kind: string;
}
```

Collapse state:
- `state.collapsed: Set<string>` — full collapse (double-click)
- `state.selectiveHidden: Set<string>` — selective hide (single-click / auto-collapse)
- `applyCollapseState()` — syncs DOM classes from both sets
- `isNodeVisible(node)` — returns false if hidden by either mechanism

Existing helpers:
- `getChildren(id)` — returns direct child node IDs from `state.links`
- `getDescendants(id)` — returns all descendant IDs recursively
- `drawLinks()` — redraws SVG links after position changes
- `updateBoardTransform()` — applies viewport transform
- `autoSave()` — debounced save after state changes

Board dimensions: 2400 x 1500 (from `.board` CSS).
Default node widths: source 318px, analysis 318px, option/generated 318-360px.

<task type="auto">
  <name>Task 2: Implement tree layout algorithm and collapse logic</name>
  <files>public/app.js</files>
  <action>
Implement `arrangeCanvasLayout()` in `public/app.js` with the following behavior:

1. **Collapse un-generated nodes:**
   - Iterate over all nodes in `state.nodes`.
   - For any node where `id.startsWith("option-")` and `!node.generated`, add its ID to `state.selectiveHidden`.
   - Call `applyCollapseState()` and `updateCollapseControls()`.

2. **Build visible tree structure:**
   - Collect all visible nodes using `isNodeVisible(node)`.
   - Compute depth (level) for each visible node:
     - `source` is depth 0.
     - `analysis` is depth 1.
     - For other nodes, depth = parent depth + 1. Find parent via `state.links.find(l => l.to === id)`.
   - Group visible nodes by depth into a `Map<number, string[]>`.

3. **Compute tree layout positions:**
   Use a layered (Sugiyama-style) layout:
   - Horizontal layers: each depth gets a column.
   - Column spacing: 420px (enough for 360px generated nodes + link gap).
   - Start X: 96px for depth 0 (matches source node default).
   - Row spacing within a column: node height + 40px vertical gap.
   - For each depth column, center the column's nodes vertically around the midpoint of the board (750px), or stack from top with a 120px top margin.
   - Specifically:
     ```
     const COLUMN_GAP = 420;
     const ROW_GAP = 40;
     const START_X = 96;
     const START_Y = 120;
     for each depth (sorted ascending):
       const x = START_X + depth * COLUMN_GAP;
       const nodesAtDepth = depthGroups.get(depth);
       const totalHeight = nodesAtDepth.reduce((sum, id) => sum + state.nodes.get(id).height + ROW_GAP, 0) - ROW_GAP;
       let y = START_Y;
       // If totalHeight fits within board height (1500), center vertically:
       if (totalHeight < 1500 - START_Y * 2) {
         y = (1500 - totalHeight) / 2;
       }
       for each nodeId in nodesAtDepth:
         targetPositions.set(nodeId, { x, y });
         y += state.nodes.get(nodeId).height + ROW_GAP;
     ```
   - Ensure positions stay within board bounds (clamp x to [0, 2400 - width], y to [0, 1500 - height]).

4. **Animate to new positions:**
   - Implement `animateNodesToPositions(targetPositions, duration = 400)`:
     - For each node in `targetPositions`, read current `node.x`, `node.y`.
     - Use `requestAnimationFrame` with a timestamp-based ease-out-cubic interpolation:
       ```js
       const ease = t => 1 - Math.pow(1 - t, 3);
       ```
     - On each frame, update `node.element.style.left/top` and `node.x/node.y`.
     - After the final frame, call `drawLinks()` and `autoSave()`.
     - During animation, temporarily add a CSS class `is-arranging` to the board to disable pointer events on nodes (prevents drag conflicts).

5. **Edge cases:**
   - If no visible nodes besides source/analysis, still position them cleanly.
   - If a node has no parent link (orphan), place it at depth 0 or 1 based on its id.
   - Preserve `state.view` (do not reset zoom/pan).

6. **Do NOT** save an "arranged" flag to the database. The positions are saved as part of normal auto-save, but there is no persistent "arranged" state.
  </action>
