## Routing

```
/history/?view=library
```

### App.tsx Changes

Update `getCurrentView()` to handle `view=library`:

```typescript
function getCurrentView() {
  const params = new URLSearchParams(window.location.search);
  const view = params.get("view");
  if (view === "settings") return "settings";
  if (view === "library") return "library";
  return "history";
}
```

## Data Flow

### API Integration

**Endpoint:** `GET /api/materials?q={query}&sort={sort}`

**Response:**
```typescript
interface MaterialsResponse {
  ok: boolean;
  items: MaterialItem[];
  total: number;
}
```

**MaterialItem (from Prisma):**
```typescript
interface MaterialItem {
  id: string;
  fileName: string;
  mimeType: string;
  fileSize: number;
  hash: string;
  filePath: string;
  addedAt: string; // ISO 8601
  updatedAt: string; // ISO 8601
}
```

### Thumbnail URL

Image thumbnails: `/api/materials/{id}/thumbnail` (not yet implemented — Phase 14 will need a thumbnail serving endpoint or use the filePath directly)

**Decision:** For Phase 14, serve images via a new `GET /api/materials/:id/file` endpoint that streams the file from disk. Non-image files show type icons instead.
