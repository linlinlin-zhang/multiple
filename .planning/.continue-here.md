## Key Findings (from root-cause analysis)

### 1. Chat→Canvas Broken
- `server.js:96-137` defines `CANVAS_TOOL_TYPES` which does NOT include `create_note`, `create_plan`, `create_todo`, `create_weather`, `create_map`, `create_link`, `create_code`.
- `normalizeVoiceActions` filters against `VOICE_ACTION_TYPES = new Set(CANVAS_TOOL_TYPES)`, silently dropping rich node actions.
- **Fix**: Replace `CANVAS_TOOL_TYPES` with import from `src/prompts/shared.js CANVAS_ACTION_TYPES` (single source of truth).

### 2. Replies Too Short
- `src/prompts/chat.js:18` — "Usually 1-3 sentences for chat replies."
- `src/prompts/shared.js META_DIRECTIVES` — short-reply constraints.
- `src/prompts/chat.js:48` — JSON schema says `"reply":"short user-facing answer"`.
- `server.js:1056` — forces JSON object mode in no-thinking mode, compressing output.
- `buildChatActionSystemPrompt` overwrites the conversational system prompt with action orchestrator prompt.
- **Fix**: Remove hard length caps, set max_tokens >= 4096 default, use OpenAI tool calling so reply is free-form markdown.

### 3. No Streaming in Default Mode
- `public/app.js:3411` — streaming only when `effectiveThinkingMode === "thinking"`.
- **Fix**: Enable SSE streaming for no-thinking mode too.

### 4. No Markdown Rendering
- `public/app.js:5722` — `text.textContent = message.content` (plain text).
- **Fix**: Render with micromark + DOMPurify as sanitized HTML.
